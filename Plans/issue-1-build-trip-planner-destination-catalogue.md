# Issue #1: Build trip planner destination catalogue

## Original Issue

Build a static Astro site that serves as a modern holiday brochure / destination catalogue for Darren and Anthony to decide where to go on holiday. UK-based travellers.

### Destinations
- At least 30 destinations, broken down by city
- If a city is worthy of a trip on its own, make it a standalone trip entry
- Mix of European destinations (we're in the UK) plus worldwide (Canada, Japan, etc.)

### Per Destination Content
- **Summary** — sell the destination, make it compelling
- **Things to do** — activities, experiences, day trips
- **The vibe** — what's the atmosphere/culture like
- **Things to see** — landmarks, natural wonders, must-visit spots
- **Allergen safety** — how the country/city fairs for nut allergy safety and vegetarian food availability. Darren has a nut allergy and is vegetarian, so this is critical safety info
- **Ideal time to visit** — best months/season
- **Recommended trip length** — how many days/nights
- **Rough hotel cost** — per night estimate in GBP
- **Rough flight cost** — return from UK in GBP

### Photos
- Use real photos for each destination (Unsplash or similar free-to-use sources)
- Hero images for each destination
- Supporting images for key attractions

### Design
- Modern, visually stunning brochure aesthetic
- Clean typography, generous whitespace
- Responsive (desktop + mobile)
- Easy to browse and compare destinations
- No AI slop — content should feel authentic and well-researched

### Technical
- Static Astro site
- No backend needed, pure static catalogue
- Fast, optimised images
- Good SEO basics

### Dietary & Allergy Context
- Darren has a **nut allergy** (tree nuts) — this is a genuine safety concern, especially in Asian cuisine countries
- Darren is **vegetarian**
- Content should honestly assess how easy/difficult each destination is for these dietary needs
